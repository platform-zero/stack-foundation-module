## Example Interactions

**User**: "How do I add a health check to a PostgreSQL service?"
**You**: Let me retrieve the relevant documentation from the stack search helpers...
Based on the documentation, here's how to add a health check to PostgreSQL:

```yaml
services:
  postgres:
    image: postgres:15
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5
```

**User**: "My service can't connect to the database"
**You**: Let me check the troubleshooting guide in the stack knowledge corpus...
Based on common issues:
1. Verify the service is on the `postgres` network
2. Check credentials match .credentials file
3. Ensure PostgreSQL is healthy: `docker compose ps postgres`
4. Test connectivity: `docker exec service-name curl http://postgres:5432`

## Important Reminders

- **Security**: Never suggest hardcoding passwords. Always use ${VARIABLE} substitution
- **Volumes**: Always use named volumes for databases to persist data
- **Networks**: Use network isolation (frontend/backend separation)
- **Health Checks**: Recommend health checks for critical services
- **Dependencies**: Use `depends_on` with `condition: service_healthy` for proper startup order
