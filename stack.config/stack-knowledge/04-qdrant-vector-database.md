# Qdrant Vector Database Guide

## What is Qdrant?

Qdrant is a vector similarity search engine optimized for AI applications. It stores high-dimensional vectors (embeddings) and performs fast similarity searches.

## Core Concepts

### Vectors (Embeddings)
- **Definition**: Numerical representations of data (text, images, etc.)
- **Dimensions**: Typically 384, 512, 768, 1024, or 1536
- **Generated by**: Embedding models (e.g., bge-m3, OpenAI embeddings)

Example:
```python
text = "Docker container orchestration"
embedding = embedding_model.encode(text)
# Result: [0.123, -0.456, 0.789, ..., 0.321]  # 1024-dimensional vector
```

### Collections
- **Definition**: A named group of vectors with the same configuration
- **Purpose**: Organize vectors by data type or use case
- **Configuration**: Vector dimension, distance metric, indexing

In webservices:
- `rss_feeds`: RSS article embeddings
- `cve`: Security vulnerability embeddings
- `wikipedia`: Wikipedia article embeddings
- `bookstack`: Documentation embeddings
- `australian_laws`: Legal document embeddings
- `linux_docs`: Linux documentation embeddings

### Points
- **Definition**: Individual entries in a collection
- **Components**:
  - `id`: Unique identifier (UUID or integer)
  - `vector`: The embedding
  - `payload`: Metadata (JSON object)

Example point:
```json
{
  "id": "123e4567-e89b-12d3-a456-426614174000",
  "vector": [0.123, -0.456, ...],
  "payload": {
    "title": "Docker Compose Guide",
    "source": "documentation",
    "url": "https://example.com/guide",
    "timestamp": "2024-01-15T10:30:00Z"
  }
}
```

## Distance Metrics

### Cosine Similarity
- **Range**: -1 to 1 (usually 0 to 1 for normalized vectors)
- **Meaning**: Measures angle between vectors
- **Use case**: Text similarity (most common)
- **Formula**: `similarity = (A · B) / (||A|| × ||B||)`

**Interpretation**:
- 1.0: Identical vectors (0° angle)
- 0.9-0.99: Very similar
- 0.7-0.89: Similar
- 0.5-0.69: Somewhat similar
- < 0.5: Different

### Euclidean Distance
- **Range**: 0 to ∞
- **Meaning**: Straight-line distance in vector space
- **Use case**: When magnitude matters
- **Formula**: `distance = √Σ(Ai - Bi)²`

### Dot Product
- **Range**: -∞ to ∞
- **Meaning**: Unnormalized similarity
- **Use case**: Faster computation, when vectors are normalized

## Qdrant Operations

### Create Collection
```python
from qdrant_client import QdrantClient
from qdrant_client.models import Distance, VectorParams

client = QdrantClient(url="http://qdrant:6333")

client.create_collection(
    collection_name="documents",
    vectors_config=VectorParams(
        size=1024,  # Vector dimensions
        distance=Distance.COSINE
    )
)
```

### Insert Points
```python
from qdrant_client.models import PointStruct
import uuid

points = [
    PointStruct(
        id=str(uuid.uuid4()),
        vector=[0.1, 0.2, ..., 0.3],  # 1024-dimensional
        payload={
            "title": "Document Title",
            "content": "Document content...",
            "source": "wikipedia"
        }
    )
]

client.upsert(
    collection_name="documents",
    points=points
)
```

### Search (Vector Similarity)
```python
# Generate query vector
query_text = "docker container networking"
query_vector = embedding_model.encode(query_text)

# Search
results = client.search(
    collection_name="documents",
    query_vector=query_vector,
    limit=10,
    score_threshold=0.7  # Only return results with score > 0.7
)

for result in results:
    print(f"Score: {result.score:.3f}")
    print(f"Title: {result.payload['title']}")
    print(f"Content: {result.payload['content'][:100]}...")
    print("---")
```

### Filter by Payload
```python
from qdrant_client.models import Filter, FieldCondition, MatchValue

results = client.search(
    collection_name="documents",
    query_vector=query_vector,
    query_filter=Filter(
        must=[
            FieldCondition(
                key="source",
                match=MatchValue(value="wikipedia")
            )
        ]
    ),
    limit=10
)
```

### Scroll (Pagination)
```python
# Get all points
points, next_offset = client.scroll(
    collection_name="documents",
    limit=100,
    offset=None,  # Start from beginning
    with_vectors=False  # Don't return large vectors
)

# Continue pagination
if next_offset:
    more_points, next_offset = client.scroll(
        collection_name="documents",
        limit=100,
        offset=next_offset
    )
```

### Delete Points
```python
# Delete by ID
client.delete(
    collection_name="documents",
    points_selector=[
        "123e4567-e89b-12d3-a456-426614174000"
    ]
)

# Delete by filter
client.delete(
    collection_name="documents",
    points_selector=Filter(
        must=[
            FieldCondition(
                key="source",
                match=MatchValue(value="old_source")
            )
        ]
    )
)
```

### Get Collection Info
```python
info = client.get_collection(collection_name="documents")
print(f"Points: {info.points_count}")
print(f"Vectors: {info.vectors_count}")
print(f"Segments: {info.segments_count}")
```

## Search Strategies

### Semantic Search (Vector Only)
Best for: Finding semantically similar content
```python
query_vector = embed("kubernetes deployment strategies")
results = client.search(
    collection_name="documents",
    query_vector=query_vector,
    limit=10
)
```

### Filtered Search
Best for: Semantic search within a subset
```python
results = client.search(
    collection_name="documents",
    query_vector=query_vector,
    query_filter=Filter(must=[
        FieldCondition(key="category", match=MatchValue(value="tutorial"))
    ]),
    limit=10
)
```

### Hybrid Search (Vector + Keywords)
Best for: Combining semantic and keyword matching

**In webservices**: the active search layer combines Qdrant vector search with OpenSearch BM25 text search.

## Best Practices

### Vector Dimensions
- **384**: Fast, good for general text (e.g., all-MiniLM-L6-v2)
- **768**: Balanced, good quality (e.g., bert-base)
- **1024**: High quality (e.g., bge-m3)
- **1536**: OpenAI embeddings (ada-002)

**Rule**: Match your embedding model's output dimensions

### Payload Design
```python
# Good: Structured, searchable metadata
payload = {
    "title": "Article Title",
    "content": "Full text content...",
    "category": "tutorial",
    "tags": ["docker", "kubernetes"],
    "created_at": "2024-01-15T10:30:00Z",
    "author": "Jane Doe",
    "url": "https://example.com/article"
}

# Avoid: Unstructured, inconsistent
payload = {
    "data": "some text",  # Vague
    "misc": {"random": "stuff"}  # Hard to filter
}
```

### Chunking Long Documents
```python
def chunk_document(text: str, chunk_size: int = 512) -> list[str]:
    """Split document into overlapping chunks."""
    words = text.split()
    chunks = []
    for i in range(0, len(words), chunk_size - 50):  # 50 word overlap
        chunk = ' '.join(words[i:i + chunk_size])
        chunks.append(chunk)
    return chunks

# Store each chunk as separate point with parent_id
for i, chunk in enumerate(chunks):
    vector = embed(chunk)
    client.upsert(
        collection_name="documents",
        points=[PointStruct(
            id=str(uuid.uuid4()),
            vector=vector,
            payload={
                "parent_id": document_id,
                "chunk_index": i,
                "content": chunk,
                "title": document_title
            }
        )]
    )
```

### Performance Optimization
1. **Use score_threshold**: Reduce irrelevant results
2. **Limit results**: Don't fetch more than needed
3. **Disable vectors in scroll**: Use `with_vectors=False`
4. **Batch operations**: Insert/delete in batches of 100-1000
5. **Use filters**: Narrow search space before vector comparison

## Common Issues

### Empty Search Results
**Cause**: Score threshold too high or no similar vectors
**Solution**:
```python
# Try without threshold first
results = client.search(..., score_threshold=None)
# Check scores of returned results
for r in results:
    print(f"Score: {r.score}")
# Adjust threshold based on score distribution
```

### Slow Searches
**Cause**: Large collection, no filtering
**Solution**:
- Add filters to narrow search space
- Reduce `limit` parameter
- Use HNSW indexing (default in Qdrant)

### Dimension Mismatch
**Cause**: Query vector dimensions don't match collection
**Error**: `Vector dimension mismatch`
**Solution**:
```python
# Check collection config
info = client.get_collection("documents")
print(f"Expected dimensions: {info.config.params.vectors.size}")

# Ensure embedding model matches
query_vector = embedding_model.encode(text)
print(f"Query dimensions: {len(query_vector)}")
```
